Управление мамботами: Новый / Изменить

Мамбот - маленькая, ориентированная на определенную задачу функция, которая производит определенные действия с содержимым, перед его отображением и манипулированием им. Они могут быть как простыми функциями, заменяющими исходный код текстом, так и полными библиотеками от посторонних разработчиков (например, WYSIWYG-редакторы).

Страница "Управление мамботами: Создание / Изменение" позволит вам редактировать мамботы сайта или создавать новые.

Значки панели инструментов:
Сохранить: Сохранение изменений в мамботе и возврат в "Управление мамботами".

Применить: Сохранение изменений в мамботе без возврата на предыдущую страницу.

Отмена: Отмена все сделанных изменений в мамботе. Мамбот останется "незаблокированным".


Детали мамбота:

Название: Название мамбота. Под таким именем мамбот будет отображен в списке мамботов сайта. Никак не влияет на ваш сайт.

Папка: Показывает, в какой папке находится мамбот.

Файл мамбота: Файл мамбота на сервере.

Порядок мамбота: Выберите порядок расположения мамботов в общем списке.

Уровень доступа: Уровень доступа для просмотра содержимого мамбота. Общий, Участники или Специальный.

Опубликован: Опубликование или сокрытие мамбота на сайте.

Описание: Описание функций мамбота.


Параметры:
Если есть настраиваемые параметры мамбота, то они будут показаны в этой части экрана.


Мамботы сайта, идущие в стандартной комплектации Joomla:

Изображение MOS:

Использование: { mosimage }

Этот мамбот заменяет тэг { mosimage } изображением.  Вы можете вставлять этот тэг в любом месте новости или статьи.  Это нужно сделать при создании или редактировании новости/статьи.  После вставки тэга, вы можете выбрать изображение, которым он будет заменен. Выберите изображение на вкладке "Изображения".


GeSHi:

Отображает форматированный код, основанный на скрипте подсветке кода GeSHi.

Да, это усовершенствованная версия мамбота mosCode, поддерживает множество языков и автоматически определяет, какой язык подсвечивать.

Использование: Используйте тэг "pre" и добавьте в него описание языка программирования для правильной подсветки кода. Например:

< pre lang="xml" >здесь ваш код ... < /pre >

Текущая версия мамбота GeSHi поддерживает следующие языки, готовые для использования:

  • CSS => используйте lang="css"
  • Html4 Strict => используйте lang="html4strict"
  • JavaScript => используйте lang="javascript"
  • PHP => используйте lang="php"
  • PHP Brief => используйте lang="php-brief"
  • SQL => используйте lang="sql"
  • XML => используйте lang="xml"

Если вы нуждаетесь в поддержке дополнительных языков программирования, то Вы можете загрузить полный пакет с сайта разработчика мамбота: http://qbnz.com/highlighter/ . Он содержит около 30 языков программирования. Вы должны будете загрузить файл "newLanguage.php" из архива на сервер в папку "\mambots\content\geshi\geshi".

Начиная с Mambo версии 4.5.2.1, этот мамбот имеет несовместимость с мамботом маскировки e-mail.


Включение наследования мамботов: 

Этот мамбот поддерживает совместимость мамботов, написанных еще для Mambo версии 4.5 с более новыми версиями.
По умолчанию этот мамбот неопубликован; если вы имеет проблемы со старыми версиями мамботов, опубликуйте этот мамбот.


Поддержка кода (moscode):

Отображает в объектах содержимого форматированный код. Нет синтаксической подсветки. Код будет отформатирован в Courier.

Использование: { moscode }... { /moscode }

Например:
{ moscode }<body onload="winManage(850,700,true);"><a name="top"></a><div><div id="wrapper"></div></div>{ /moscode }

Рейтинг MOS:

Добавляет возможность голосования в объектах содержимого.

Для того, чтобы голосование было видно, необходимо опубликовать мамбот и вы должны включить опцию "Рейтинг/Голосование:" в "Глобальной конфигурации" -> "Содержимое".

После этого пользователи получат возможность голосовать за новости/статьи по 5-бальной шкале. Они смогут проголосовать только один раз.

Максировка Email: 

Этот мамбот скрывает все адреса e-mail на вашем сайте. При редактировании мамбота, вы можете настроить параметр  -> Режим работы:
  • Просто текст (не ссылка) отображает e-mail как простой текст, без ссылки.
  • Ссылка для отправки почты показывает e-mail вместе с ссылкой для отправки почты, но используя специфический адрес.

Позиции загрузки модуля: 

Загрузка модулей в определенной позиции в пределах содержимого.

Использование: {mosloadposition user1}, где 'user1' - это одна из позиций для ваших модулей.

Это означает, что вы можете загрузить любой модуль в любую позицию вашего содержимого без правки кода шаблона в файле index.php.

Параметр позволяет настроить код, определяющий расположение модуля в позиции:
  • В таблице - в столбец
  • В таблице - горизонтально
  • В тэгах DIV
  • Без вложения, без обработки

Разбиение на страницы MOS (mospagebreak): 

Добавляет функцию создания оглавления содержимого.

Использование:
{ mospagebreak } или
{ mospagebreak title=Название страницы } или
{ mospagebreak heading=Первая страница } или
{ mospagebreak title=Название страницы&heading=Первая страница } или
{ mospagebreak heading=Первая страница&title=Название страницы }

Примечание: Не должно быть пробелов между фигурными скобками и текстом внутри них.

Новый класс 'contenttoc' CSS используется для оформления оглавления страниц.



Простой редактор HTML:

Загрузка базовых полей для ввода текста.


WYSIWYG-редактор TinyMCE:

TinyMCE - визуальный редактор HTML, основанный на Javascript.


Кнопка вставки изображения MOS в редакторе:

Предоставляет возможность с помощью кнопки вставить изображение {mosimage} в окно редактирования содержимого.


Кнопка разрыва страницы MOS в редакторе:

Позволяет кнопкой вставлять разрыв страницы {mospagebreak} в окне редактирования содержимого.
